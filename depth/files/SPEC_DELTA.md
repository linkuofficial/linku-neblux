### C5. 公式符號懸浮註釋(發布前提)

2026-07-11 凜空指示:**每頁公式的特殊符號都要有懸浮註釋**,四頁通用,新頁一律照此。

**markup 契約**(單一事實來源 = 頁內詞彙表,不另建 JS 字典):

```html
<!-- 公式中每個特殊符號:指標 hover 提示,不設 tabindex(見無障礙) -->
<span class="sym <colour>" data-sym="KEY" data-tip="一句話註釋">符號</span>
<!-- 詞彙表:dt[data-sym] 是鍵盤/AT 的正式互動停點(模組會補 tabindex/role),觸控 ≥44px -->
<dl class="symbol-gloss">
  <dt data-sym="KEY"><span class="sym <colour>">符號</span></dt><dd>完整說明</dd>
</dl>
<p class="sym-explainer" aria-live="polite">把游標移到算式的符號上(手機點一下),看它對應畫面上的哪個東西。</p>
<div class="sym-tip" role="tooltip" hidden></div>
```

**行為**:hover(指標)或 focus(鍵盤,於詞彙表 dt)顯示浮動 `.sym-tip`(內容取 `data-tip`,無則退回對應 `<dd>` 文字),並 **cross-highlight** 所有同 `data-sym`(公式符號 + 詞彙表 `dt`)加 `.is-active`;手機點擊 = pin,Esc / 點區塊外 = unpin。

**無障礙**:公式符號本身只作指標 hover 提示、**不設 `tabindex`**(避免公式塞入一串冗餘 tab 停點);鍵盤/AT 走詞彙表——`dt[data-sym]` 由模組補 `tabindex="0"` + `role="button"` + `aria-label`(= tip 文字) + focus 可見,觸控 ≥44px,是唯一且乾淨的 tab 停點。tip 文字/可見 `<dd>` 為**非色彩通道**,符號另以 `.is-active` 的底線/外框標示(不只變色);停用 JS 時 `<dl>` 即完整可讀 fallback。

**實作分層**:
- 通用行為由共用模組 **`depth/sym-tooltip.js`** 提供(`data-tip` 驅動、自初始化、CSP-safe 外置 module;停用 JS 時詞彙表 `<dl>` 即完整可讀 fallback)。新頁只要引入該模組 + 遵守上面 markup,即得完整 tooltip。
- 符號同時要**連動 canvas 高亮**的頁,設 `window.__nebluxSymHook = (sym) => {…}`(於模組載入前)提供 per-page render hook。
- 現況債:`s-plane`/`transformer` 為本規範前的 **inline 參考實作**(tooltip 邏輯寫在各自 `*.js`、tip 文字存 JS `GLOSS` 物件、尚未用 `data-tip`);行為已符合本規範,待遷移到共用模組 + `data-tip` 契約(記入 BACKLOG,遷移時逐頁瀏覽器驗證)。

### C6. 三重對應明講:畫面顏色＝公式符號＝控制標籤(發布前提)

2026-07-11 凜空指示。核心巧思是「同色＝同一個東西」,但**不得只靠顏色剛好一樣來暗示**——三處都要明講,讓對應看一眼就懂、不必推理:

1. **每個參數固定一色**(頁內 CSS token,如 `--amp`/`--zeta`/`--q`),同一參數在滑桿、公式、畫面三處永遠同色。
2. **滑桿/控制標籤**:標籤文字前放實色小色塊 `<i class="chip chip-{param}" aria-hidden="true"></i>`(11px,圓角 3px)。
3. **畫面(canvas)**:該參數對應的線條/元素**末端放同色標籤**——色塊＋名字(如「A 振幅」),深色底墊片,不能只有一個裸字母。座標夾制避免出界。
4. **圖例一律 HTML**(`.canvas-legend` + `.chip`),不用 canvas 小字——可放大、可斷行、可被 AT 讀到。曲線身分(目標/合成)也算參數,一樣進圖例。
5. **關鍵現象直接在畫面上標註**:教學重點(如 Gibbs 突起、過衝峰值)用小圈/箭頭＋短籤指出「該看哪裡」,數值隨互動即時更新;籤文隨情境進化(例:N≥4 時改「這個尖角不會消失(+9%)」)。
6. 色彩仍**不得為唯一通道**:虛實線、形狀、文字籤並用(C1 不變)。
7. **軸標用真實標籤,不用索引數字**:熱圖/矩陣的行列軸直接寫 token 字、單位或名稱,不能讓使用者心算「第 2 格=cat」。窄版面擺不下就縮字級或斜排,不得退回數字。
8. **熱圖對比要拉開**:亮度對「全矩陣最大值」正規化＋gamma 拉伸(最暗更暗、最亮=滿色),並在每格疊一個淡數字作第二通道——色弱使用者也分得出深淺。

### C7. 先玩後式的頁面結構(發布前提)

2026-07-11 凜空指示。教學哲學=「先玩、再觀察、想懂再看式子」,頁面結構要把這個順序**視覺上逼出來**:

1. **生活比喻入口**:intro 在 `.deck` 之後放 `figure.metaphor`——一張小 inline SVG(340×84 viewBox 基準,用頁內 token 上色,參數色可預告)＋一句話生活比喻(figcaption)。先用圖像建立直覺,再進互動。比喻要日常(音響/合唱/減速丘/吵雜房間),不得用術語。
2. **步驟鏈 kicker**(2026-07-11 第三輪更新):標準四步=`1 先動手玩` → `2 觀察` → `3 哪裡用得到` → `4 寫成式子`;**公式永遠是最後一步**(「連結」不編號)。互動只涵蓋主題切片時,在「觀察」後插入選配步驟「拉遠看」(見第 9 條),後續步驟順延編號。
3. **公式區預設收合**:形式化內容包進 `<details class="formal-details">`,`<summary>` 內放該區 `<h2>` ＋邀請語 `.summary-hint`(如「玩過了?打開來看」)。原生 `<details>`:no-JS 也能開合,收合狀態就是預設的「先別被公式嚇到」。
3b. **「哪裡用得到」段(必備)**:觀察之後、公式之前,`.usecases` 區放 3–4 條真實生活實例(`ul.usecase-list`,粗體詞＋一句白話,可嵌參數色符號);例子必須屬實且日常(音量鍵、EQ、避震、ChatGPT),不虛構數據。目的:讓使用者在看公式前先知道「這東西通常用在哪」。
4. **不做遊戲化**:不做解鎖、進度條、成就(牴觸品牌「安靜的驚奇」)。順序靠結構與收合暗示,不靠機制強制。
5. **參數↔畫面連動強調**:多圖連動的頁(如 s-plane 左點右線),滑桿 input 時讓因果兩端**同拍脈衝一次**(~600ms 淡出;`prefers-reduced-motion` 時停用,平時狀態不變)。
6. **狀態/模式類概念做成可點縮圖列**(如四種阻尼個性):每個模式一張迷你曲線 SVG＋名字＋條件,點了直接把參數設成該模式代表值,目前所在模式亮框(`aria-pressed` 同步);列上方放一行 microcopy(「點一張卡,滑桿直接跳到那種狀態」)+ hover 邊框,讓可點性顯而易見。文字條列不足以讓人「一眼看到 N 種個性長什麼樣」。
7. **熱圖/矩陣類互動加白話層**:hover(指標)到資料格時,浮出「白話句＋數值」(如「『it』看『cat』＝0.83」),格子加白框;觸控端以既有點選互動為主,浮籤為增強。
8. **增量顯形**:「參數 +1」型互動(疊一顆、加一層),參數改變時把「剛加入的那個成分」用專屬色單獨畫出,短暫顯示(~0.9s)後淡出融入總和;圖例加一項說明。`prefers-reduced-motion` 時不做淡出動畫,改靜態顯示片刻後單次重繪隱藏。
9. **拉遠看(選配步驟)**:互動只涵蓋主題的一個切片時(如 Transformer 頁只拆一顆注意力頭),在「觀察」之後加「拉遠看」段:一張方塊流程小圖(SVG,互動涵蓋的方塊用主色標出＋「你剛玩的就在這格」註記)＋逐條白話說明其餘組件;誠實標注互動範圍,不讓切片被誤讀為全貌。
10. **參數改變的立即回饋**:連動脈衝(第 5 條)第一幀直接同步繪製(不等下一個 rAF),拉桿當下就看得到。

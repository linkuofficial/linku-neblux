# SPEC 差異清單(取代 DEPTH_PAGE_SPEC v1.0 PROPOSAL)

## A. 保留條款(自 v1.0 移入,效力不變)

1. **答案先行(BLUF)**:每個 H2 下第一段為 150–200 字內、去代名詞、可獨立抽取的完整陳述。
2. **資訊增益一票否決**:每頁至少一項現有繁中供給沒有的東西(推導/對照表/實測數字/互動),否則不發布。
3. **AI 產出=未驗證草稿**:一切 AI 輔助技術內容不得直接進發布版。
4. **台灣術語標準表**(隨規格維護;首見附英文原文):快取(非緩存)、向量(非矢量)、穩健(非魯棒)、記憶體(非內存)、顯示記憶體/VRAM(行文可括注顯存)、演算法(非算法)、資料(非數據,固定詞除外)、函式(非函數,數學函數除外)、迴圈/陣列/佇列/執行緒/原始碼/軟體/硬體。
5. **漸進增強**:停用 JS 時頁面必須是完整可讀文章;每個互動區塊附靜態截圖+一段文字描述其展示的現象。
6. **一互動一問題**:開工前寫下「這個互動讓使用者親手發現什麼」,答不出即砍。
7. **英文獨立寫作**:不做機械直譯;英文版僅在理解閘門通過後製作。
8. **Living Document**:footer 版次+changelog;實質更新同步 `dateModified`;時效敏感內容標「截至 YYYY-MM」。
9. **OG 圖**:互動實際截圖,不用 logo。
10. 事實密度、引用格式統一、一手來源要求:不變。

## B. 修正條款

1. **語域**:刪除「工程師對工程師」。改為**可及性優先分層**——第一層供零先備知識的探索者(品牌規範受眾),工程推導、公式、效能數據放進階摺疊區。參考語域:Ciechanowski(零先備可讀而工程師敬重),非長文技術指南體。
2. **fan-out**:子問題收回頁內(H2 真實問句形式保留);僅當子問題形成獨立搜尋意圖、獨立推導或工具價值時才拆 URL,拆分需個案核可。
3. **結構化資料**:保留 `Article`(author/datePublished/dateModified)+ `DefinedTerm` + canonical + hreflang;`FAQPage` 預設移除(2023-08 起富摘要僅限權威站點,展示收益趨零);結構化資料必須忠實反映頁面可見內容;一律稱「語意衛生」,不稱 GEO 保證。
4. **分析工具**:刪除 Amplitude 全部條款(含五事件)。理由:違反 Neblux 隱私模型(禁 cookie/個人追蹤)與 CSP `'self'`。未來量測走同源聚合,另開 brief。
5. **資料源**:刪除一切 nodes.json 引用,改為 `depth_manifest.json` 掛接 `all_nodes.json`(見 V03_AMENDMENTS 第 2 節)。
6. **效能條款改寫**:原「互動響應 <16ms=60fps」混淆單幀預算與輸入延遲,改為——在指定低階測試機(建議:3 年前中階 Android 機型,型號由凜空指定)上實測:互動期間 frame time p95 < 16.7ms;互動關鍵路徑無 >50ms long task;單頁初載 JS(gzip)< 150KB 維持。
7. **HTML 體積**:200KB 上限保留,定性為效能預算。索引事實備註:Googlebot 對 HTML 的索引抓取上限為 2MB(2026-02 官方文件),超限在截點停止、僅索引已下載部分,故關鍵 meta/JSON-LD 置於文件前段——本站遠低於門檻,此為衛生慣例非風險項。

## C. 新增條款

### C1. 無障礙(發布前提)
- 鍵盤:所有互動控制項可 Tab 到達、可操作(滑桿支援方向鍵),focus 狀態可見。
- Canvas/SVG 互動:提供文字替代(aria-label + 鄰接說明文字);互動結果的關鍵數值同步以文字呈現。
- `prefers-reduced-motion`:尊重並提供靜態替代。
- 色彩:不作為唯一資訊通道(輔以形狀/標籤);文字對比 ≥ 4.5:1。
- 觸控目標 ≥ 44px(保留)。

### C2. 驗證(發布前提)
- 數學核心抽為純函式,附單元測試(含邊界條件)。
- E2E:no-JS 可讀性、CSP 無違規、手機+桌機各一輪。
- Depth validator 納入 `npm run verify`,CI 必跑。

### C3. 技術審查協議(發布前提,一票否決)
1. **claim-source 對照表**:頁面每條技術陳述(定義/公式/數值/因果)逐字列入,對照一手來源(含式號/頁碼),用 `CLAIM_SOURCE_TABLE_TEMPLATE.md`。
2. **公式適用條件**:每條公式標明變數定義與成立前提。前車之鑑:KV Cache 顯存公式漏乘序列長度與 batch size(每 token 量被寫成總量)。
3. **可重跑驗證**:關鍵數值附可重跑的測試檔或 notebook,路徑寫入對照表;與原論文圖表或官方實作交叉比對。
4. **逐符號自述**:凜空能不看稿解釋每條公式的每個符號,做不到的段落返工或刪除。
5. **外部審查(旗艦頁建議)**:至少一位領域相關者過目(電機系教授/博班學長為現成管道);不同 AI 模型交叉檢查可用於找錯,不得替代人的審查。
6. 審查發現與修正記入製作履歷。

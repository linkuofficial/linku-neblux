# Neblux Depth 頁面製作規範 v1.0

適用範圍:所有 depth 概念頁(旗艦/標準/支撐)。
版本策略:v1.0 為先驗規範;v2.0 將於試產批履歷分析後發布,以實測數據修訂本文件第 5 節與工時預算。
標注【硬性】者為發布前提,任一不符即不發布。

---

## 1. 架構與渲染【硬性】

- **SSR/SSG 必須**:核心文字、表格、JSON-LD 必須存在於初始 HTML。禁止 client-only 渲染核心內容;互動元件以漸進增強方式掛載。
- **一概念一 URL**:`/depth/{slug}`,英文版 `/en/depth/{slug}`。頁面由 `nodes.json` 單一登錄源生成,slug 即節點 id。
- **canonical + hreflang**(zh-Hant / en)配對;sitemap 隨建置自動更新並提交 GSC。
- **漸進增強**:停用 JS 時,頁面必須仍是一篇完整可讀的文章;每個互動區塊提供靜態截圖 + 一段文字描述該互動展示的現象(這同時是爬蟲與 AI 系統讀到的版本)。
- **HTML 精簡**:初始 HTML(未壓縮)< 200KB。Googlebot 對 HTML 的索引上限為 2MB 且超限即截斷,故 meta、canonical、JSON-LD 一律置於文件前段。

## 2. 內容結構【硬性】

- **答案先行(BLUF)**:每個 H2 標題下的第一段,必須是 150–200 字內、去代名詞、可脫離上下文獨立成立的完整陳述。先結論,後推導。
- **頁面骨架**:主定義區 → 核心互動 → fan-out 子問題區(每個 H2 為一個真實問句,如「為什麼 INT8 直接截斷會讓精度崩塌?」)→ 圖譜延伸(prerequisite / related 節點連結)。
- **資訊增益門檻**:每頁至少包含一項現有繁中供給沒有的東西——完整推導、對照表、實測數字或互動。沒有就不發布。
- **事實密度**:關鍵主張附具體數字與一手來源(原論文、官方文件),引用格式全站統一。時效敏感內容(模型名、框架版本、硬體規格)標註「截至 YYYY-MM」。
- **圖譜整合**:每頁宣告 prerequisite / related,寫回 `nodes.json`;depth 頁是圖譜的 z 軸,不是孤島文章。

## 3. 術語與語言【硬性】

- **台灣術語標準表**(隨本規範維護,首見英文術語附原文,如「量化(Quantization)」):

| 標準(繁中/台灣) | 禁用(簡中慣用) |
|---|---|
| 快取 | 緩存 |
| 向量 | 矢量 |
| 穩健 | 魯棒 |
| 記憶體 | 內存 |
| 顯示記憶體 / VRAM | 顯存(行文可括注) |
| 演算法 | 算法 |
| 資料 | 數據(固定詞除外) |
| 函式 | 函數(數學函數除外) |
| 迴圈 / 陣列 / 佇列 / 執行緒 / 原始碼 / 軟體 / 硬體 | 循環 / 數組 / 隊列 / 線程 / 源碼 / 軟件 / 硬件 |

- **雙語規則**:互動核心程式碼共用,文案分層維護。英文版為獨立寫作,不是翻譯;旗艦頁雙語必備,標準頁視題材,支撐頁繁中優先。
- **語氣**:工程師對工程師。禁行銷語,禁「在當今快速發展的 AI 時代」類廢話開場——開頭第一句就進入問題。

## 4. 技術審查閘門【硬性,一票否決】

- 每一條數學公式與技術主張,發布前須對照一手來源(原論文 / 官方文件)逐條驗證。
- 公式必須標明所有變數定義與適用條件。前車之鑑:研究底稿中 KV Cache 顯存公式漏乘序列長度與 batch size——每 token 用量被寫成總量。此類錯誤上 HN 即信譽事故。
- **AI 輔助產出的一切技術內容,一律視為未驗證草稿**,不得直接進入發布版。
- 審查人:凜空本人。審查發現與修正記入該頁製作履歷。

## 5. 互動元件(v1.0 初始約定;v2.0 依履歷數據修訂)

- **元件庫優先**:動手寫新互動前先查元件庫;新元件必須以可重用形式抽離入庫,無法抽離者在履歷中說明原因。
- **初始基元清單**(Phase 0 建置骨架):
  - `slider` 參數滑桿(含即時數值顯示)
  - `preset-buttons` 情境/模式切換
  - `canvas-layers` 分層渲染(背景網格 / 資料 / 互動控制點分層)
  - `heatmap` 熱力圖
  - `draggable-point` 可拖曳點/向量
  - `timeline-scrubber` 播放 / 暫停 / 步進
  - `formula-highlight` 公式項與圖形元素聯動高亮
- **效能預算**:互動響應 < 16ms(60fps);單頁初載 JS(gzip 後)< 150KB;行動端觸控必須完整可用(拖曳目標 ≥ 44px)。
- **開源**:元件庫與頁面互動原始碼不壓縮、公開於 linkuofficial(MIT 授權),README 註明 Neblux 出處——同時服務社群信任溢價與站外實體共現。
- **一互動一問題**:開工前寫下「這個互動讓使用者親手發現什麼」。答不出來或互動與該頓悟無關,砍掉。

## 6. GEO / SEO 標記【硬性】

- **JSON-LD**(置於 `<head>`):`Article`(author、datePublished、dateModified)+ `DefinedTerm`(主概念)+ `FAQPage`(對應 fan-out 問句區)。
- **作者實體**:`author.sameAs` → GitHub(linkuofficial)等既有公開檔案,建立實體映射。
- **OG 圖**:該頁互動的實際截圖,不用 logo。
- **埋點**(Amplitude):`depth_view`、`interact_start`、`interact_engaged`(操作 ≥ 3 次或 ≥ 10 秒)、`scroll_depth`、`graph_jump`(跳轉圖譜其他節點)。

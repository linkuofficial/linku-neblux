## 0. 一句話

以「量化 → KV Cache → 邊緣推論」旗艦簇在英文社群賺連結與信譽,以子問題支撐頁佔據繁中互動真空,並以試產批(8–12 頁)的製作履歷回饋出可規模化的製作規範。

---

## 1. 戰略依據(三份研究的收斂結論)

1. **冷啟動唯一引擎是旗艦互動解釋 → 技術社群引爆 → 反向連結。** 新網域等不到搜尋權重;Ciechanowski / Nicky Case / Red Blob 模式證明少量高密度作品打 HN/Reddit 是可行的零起點路徑。
2. **GEO 是衛生標準,不是引擎。** SSR、答案先行、DefinedTerm/FAQ schema、sameAs 實體共識——全部照做,但不指望單獨帶量。AIO 引用高度依賴傳統前二十名,新網域三個月內對頭部詞無望;ChatGPT/Perplexity 引用機會留給長尾子問題段落。
3. **缺口在子問題層 + 互動形式。** 繁中主概念層供給飽和(維基、iT邦、簡體溢入),但子問題層(√d_k 的梯度意義、GQA 顯存計算、RoPE 幾何、量化異常值)供給斷裂;中階技術概念的 explorable explanation 在繁中接近真空。
4. **雙語分工是結構必需。** 英文版負責 HN/Reddit 連結獲取;繁中版吃在地化 RAG 主場優勢、術語轉換疲勞受眾與 Neblux 真實用戶。
5. **這一簇內容同時是 Linku 的品牌論述。** 量化→KV Cache→邊緣推論 = 「怎麼把大模型塞進便宜的小硬體」= Linku 公司命題本身。Neblux 獲客內容與 Linku 信譽建設在此重疊。

---

## 2. 非目標(明確不做)

- 不做頭部詞 SEO 頁(不為「什麼是 Transformer」單獨建頁搶排名;主概念只作為圖譜節點與支撐脈絡存在)。
- 不在決策閘門通過前啟動量產。
- 不發布無資訊增益的模板填充頁(規範第 2 節,一票否決)。
- 不放展示廣告、不做付費牆、不做註冊牆(社群信任是本計畫的核心資產)。
- 旗艦頁不與 capstone 衝刺週重疊排程。

## 4. 選題隊列

### 旗艦軌(依序)
1. **#3 模型量化:INT8 異常值災難與混合精度保護**(預設首發,待拍板)——建造成本最低(權重分布圖 + 位元滑桿 + 異常值保護開關),頓悟感最銳利,符合「小賭注」原則。
2. **#2 KV Cache:MHA/MQA/GQA 顯存與算力權衡**——本質是帶解釋的 VRAM/OOM 探索工具,工具比文章更易被收藏連結;模擬複雜度較高,待元件庫成形後做。
3. **#5 邊緣分散式推論:延遲與拓樸模擬**——品牌含金量最高、受眾最窄,收官之作。

### 標準頁候選
RoPE 位置編碼(#4)、推測性解碼(#6)、突觸可塑性 LTP/LTD(#9)。

### 支撐頁
旗艦與標準頁的 fan-out 子問題獨立成節點,例如:「KV Cache 大小怎麼計算」「PTQ 與 QAT 的差異」「什麼是校準資料」「因果遮罩如何在矩陣運算中實作」。優先做能連進旗艦頁的節點,強化內部連結網。

### 紅海區(只做繁中支撐頁,不投旗艦資源)
QKV 幾何(#1,英文有 Transformer Explainer/BertViz)、梯度下降地形(#10,Distill 動量文 + TF Playground)、時間複雜度(#7)、動作電位(#8,PhET 繁中版已有)。

### 動工前防呆
每題開工前花 10 分鐘搜尋確認英文世界無著名互動式在位者;若有,降級為繁中支撐頁或換題。

## 5. 階段與時程

### Phase 0 基礎工程(第 1–2 週)
SSG 管線、URL 結構、schema 基建、互動元件庫骨架(規範第 5 節的初始基元)、Amplitude 埋點、製作履歷模板就位。

### Phase 1 試產批(第 3–10 週)
- 首發旗艦動工,標準/支撐頁滾動產出。
- **每頁發布當日填製作履歷,禁止事後補填。**
- 第 5 份履歷完成後做中期檢討:元件重用率、工時預算偏差、問題分類聚類。若某類問題重複 3 次以上,立即修訂規範,不等 v2.0。

### Phase 2 規範迭代(第 10–12 週)
- 彙整全部履歷,產出 `DEPTH_PAGE_SPEC v2.0` 與元件庫 v1.0 定版。
- 計算三級頁面的實測邊際工時,評估量產閘門。

### Phase 3 規模化(有條件啟動)
僅在第 6 節閘門全數通過後啟動,且量產對象限支撐頁,保留人工技術審查與資訊增益門檻。

## 6. 量測與決策閘門(首發旗艦上線後 8–12 週評估)

### 必要條件(全過才進 Phase 3)
- GSC:試產批頁面 80% 以上被收錄。
- 至少 1 個自然反向連結(非自建、非社群個人檔案)。
- 旗艦頁互動參與率(至少操作一次互動元件)> 30%(Amplitude)。

### 輔助指標
- 每月手動引用巡檢:以目標詞各查一輪 ChatGPT / Perplexity / Google AIO,記錄是否被引用(建立巡檢表)。
- HN / Reddit 發布成績(分數、留言品質、是否被技術 KOL 轉發)。
- 停留時間、繁中/英文流量比、圖譜跳轉率(depth 頁 → 其他 Neblux 視圖)。

### 未過閘門的降級路徑
depth 內容線降級為開源專案文件的附屬品;元件庫轉為 Neblux 產品內部資產;不再對「內容作為獲客管道」獨立投資源。此路徑下試產批投入仍全額回收於產品本身。

## 7. 風險登記

| 風險 | 對策 |
|---|---|
| 技術錯誤上線(前例:研究報告中 KV Cache 公式漏乘序列長度與 batch size) | 規範第 4 節技術審查閘門,一票否決 |
| 觸發規模化內容濫用政策 | 量產限支撐頁、維持發布節奏合理、每頁資訊增益門檻 |
| 與 capstone / Lino 時間衝突 | 旗艦避開 capstone 衝刺週;每週 depth 工時上限由凜空自訂並寫入履歷對照 |
| 範圍蔓延(互動越做越大) | 每頁開工前寫一句話「這個互動讓使用者親手發現什麼」,互動只服務該頓悟 |
| 履歷淪為形式 | 模板固定欄位 + 分類編碼;中期檢討以履歷數據為唯一輸入 |

# Claim–Source 對照表模板

用途:每一個 depth 頁的每條技術陳述,發布前逐條對照一手來源並可重跑驗證。本表為技術審查閘門的核心文件,路徑寫入 manifest 的 `source_notes`。

---

## 頁面:__________(slug)|審查人:凜空|外部審查:__________(選填)

| # | 頁面陳述(逐字) | 類型 | 一手來源(含式號/頁碼/表號) | 適用條件 | 驗證方式(測試檔/notebook 路徑) | 狀態 |
|---|---|---|---|---|---|---|
| 1 | | 定義 / 公式 / 數值 / 因果 | | | | pending / verified / rejected |

類型說明:
- **定義**:名詞界定 → 對照原論文或官方文件的原始定義。
- **公式**:必須列出全部變數定義與成立前提;與原文式號逐項核對。
- **數值**:任何具體數字(精度、大小、比例)→ 附來源表號,可重跑者附腳本。
- **因果**:「A 導致 B」類主張 → 來源需直接支持因果,相關性證據須改寫措辭。

## 示範列(前車之鑑,保留於模板供警示)

| # | 頁面陳述 | 類型 | 一手來源 | 適用條件 | 驗證方式 | 狀態 |
|---|---|---|---|---|---|---|
| 例 | KV Cache 大小 = 2 × 層數 × KV 頭數 × 每頭維度 × 精度位元組 × **序列長度 × batch size** | 公式 | (待補:以推導與官方實作核對) | 標準自迴歸推論;每 token 快取 K 與 V 各一份(係數 2 由此而來) | 對照小模型實測顯示記憶體佔用 | pending |

> 警示:早期研究底稿曾漏乘序列長度與 batch size,把每 token 用量寫成總量。**任何公式入表時先問:每個維度都乘齊了嗎?單位是什麼?**

## 完成條件

- 全表無 `pending` 才可進理解測試。
- `rejected` 的陳述必須從頁面移除或改寫,不得保留原文。
- 發布後的技術指正:回本表補列,修正記入頁面 changelog 與製作履歷。

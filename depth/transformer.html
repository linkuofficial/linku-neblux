自注意力 Self-Attention

一個字的意思，是它挑著看的其他字，混出來的。

在一個句子裡，每個字都在偷偷決定：其他的字，各要看多重。看得越重的，就越多地混進它自己新的意思裡——代名詞「it」到底指誰，就是這樣被決定的。

這一頁只想讓你看見一件事：一個字更新後的意思，是它對句子裡每個字分配的注意力、加權混合出來的。

這頁只拆開「一顆注意力頭、一個固定句子」這件事。用的 Q／K／V 是為了看得清楚而手挑的固定數字，不是訓練出來的。不談多頭、位置編碼、前饋層、疊很多層、也不談訓練。

互動主體

點一個字，看它把注意力給了誰

點上面句子裡的字，看它從自己的位置望出去，注意力落在哪些字身上——最亮的那個，就是它最在意的。

注意力熱圖：每一橫列＝從某個字「看出去」，格子越亮＝看那個字越重；整列加起來剛好是 1。

從「it」看出去，最亮的是「cat」（0.83）。

觀察任務

先讓畫面回答

  1. 點「it」：七格裡哪一格最亮？它其實在看句子裡的哪個字？
  2. 再點「scared」和「hid」：它們最亮的那一格是不是都落在同一個字上？（那個字被幾個字盯著）
  3. 看每一列：不管選哪個字，一整列的亮度加起來都一樣多——注意力被分給七個字，總和不變。
  4. 拉 T：把注意力從「死盯一個」慢慢拉成「平均看每一個」。總和有變嗎？

形式化

剛剛的「多重要」，寫成三行

scoreij = (qi · kj) / √d

wij = softmaxj(scoreij) = exp(scoreij) / Σk exp(scoreik)

outputi = Σj wij · vj

把游標移到算式的符號上（手機點一下），看它對應畫面上的哪個東西。

d = 4，√d = 2。每個符號對回剛剛畫面上的東西：

qi
你點的字在找什麼（teal，對應你選的字和熱圖裡它那一橫列）。
kj
token j 提供／代表什麼（amber，熱圖上緣被看的七個字）。
scoreij
qi·kj／√d，還沒正規化的原始比對分數（選中那列七格的原始強度）。
√d
縮放係數＝2，讓 softmax 不要太快飽和；跟下面的 T 滑桿是同一件事的教學版。真實模型 d≈64 時 √d=8。
softmax
把一整列七個分數壓成加起來剛好＝1 的比例——就是下面那條長條被剛好填滿。
wij
amber，熱圖裡的一格、或長條裡的一段；看得越重就越大，每一列的和＝1。
vj
token j 傳出去的內容；這個 demo 用 v=k 簡化，真實模型會學一個獨立的 W_V。
outputi
混合完的新向量：以 it 為例，output ≈ 0.83·vcat + 0.07·vit + …，幾乎全落在 ENTITY 軸上——it 接手了 cat 的身分。

拿掉 √d 的縮放，it→cat 的分數會從 0.83 衝到 0.99：差距被放大，softmax 幾乎整個押在同一格上（飽和）。√d 就是用來擋住這件事。

# Neblux 好奇導向改造計畫 — 讓人接觸並「愛上了解新事物」

> 狀態（2026-06-25 再校正）：目標明確為 **讓人接觸新事物、並愛上「了解新事物」這件事**
> ——驅動好奇、驚奇、理解的喜悅、想繼續探索。**刻意排除**會助記憶但可能帶負面情緒的機制
> （想起測驗、間隔複習、計分、SRS）。知識圖譜＋既有 Deep Field 視覺是情感與探索的*基盤*；
> 上層放的是**好奇與愉悅的層**，不是評量層。原「邊緣 AI 7 步」續用為第一個「驚奇之旅」的內容骨架。

---

## 0. 目標與誠實取捨

- **要做到**：點燃好奇、製造「原來如此！」的瞬間、讓人享受理解、想點下一個、想分享。＝*情感與動機*。
- **刻意不做**：記憶留存、精熟、技能、考試。**移除**想起練習、間隔複習、計分、SRS——它們助記憶但會帶
  「被考」「作業」的負面情緒，趕走隨興探索的人。
- **誠實取捨**：因此本站產生*喜悅與「我懂了」的時刻*，但**不**產生長期細節記憶。這是自覺的取捨，符合目標。
- 一句話：本站是**好奇心的入口（gateway）**，不是健身房。動手與精熟仍在核心軌道（邊緣＋電機控制）。

## 1. 重新評估：讓人「愛上了解」的要素（好奇／動機／情感）

| 要素 | 機制 | 此站怎麼用 |
|------|------|------------|
| **好奇缺口 Curiosity gap** | 已知與想知之間的落差會驅動「想填補」 | 每步先拋一個誘人的問題/謎，而非答案 |
| **頓悟 Aha / insight** | 突然理解本身就有愉悅（內在獎賞） | 每步設計成交付一個小「喔！」——重framing 或意外連結 |
| **驚奇 Surprise / novelty** | 違反預期的事抓住注意、自然難忘 | 主打「你絕對猜不到 X 連到 Y」——這正是 Neblux「萬物相連」的賣點 |
| **自主 Autonomy（SDT）** | 能掌控路徑時內在動機最強 | 讓人自由漫遊、追岔路；不軌道化、不強制 |
| **勝任感 Competence（SDT）** | 「我能懂」的感覺（非靠考試） | 好的鋪陳、清晰、把難的講得可親；小小的懂＝獎賞 |
| **連結感 Relatedness（SDT）** | 與人、故事、人性面的連結 | 用人物（45 個 person 節點）、發現的戲劇、歷史故事帶概念 |
| **敘事 Narrative** | 故事天生吸引、易投入 | 透過「誰、為何、怎麼想到的」說概念，而非乾條目 |
| **美感與敬畏 Beauty / awe** | 美與壯闊帶來愉悅、開放心態 | 既有 Deep Field 視覺是情感資產；大尺度 framing 製造敬畏 |
| **生動具體例 Vivid examples** | 把抽象變可感、可親 | 每概念配一個鮮活、貼近生活的例子 |
| **個人相關 Relevance** | 「干我何事」用情感回答 | 接到人本來就好奇/在乎的東西 |
| **心流 Flow** | 難度恰好＋內容本身給回饋 | 漸進、不過載；理解「卡進去」的順暢感 |
| **意外發現 Serendipity / 兔子洞** | 偶遇未知的驚喜 | 鼓勵追岔路；連結＝特性不是缺點 |
| **低風險玩樂 Low-stakes play** | 無評分、無失敗、無壓力 | 全程無對錯計分（這就是你要保護的） |
| **分享愉悅 Shareable wonder** | 「我一定要給人看這個」 | 每個驚奇可分享（URL／截圖） |

**已從上一版移除**：想起測驗、即時對錯計分、間隔複習、SRS、精熟分數、卡片。
**保留但重framing**：原「預測」→ 改成*無壓力的好奇鉤子*（拋問題引發想知道，不檢查對錯）；
原「精熟地圖」→ 改成 ***漫遊足跡**地圖*（去過/點亮的節點，是好奇旅程的紀念，不是掌握度評分）。

> 注意：移除測驗*不等於*退回「被動閱讀」（探索器的老問題）。主動成分換成了
> **好奇鉤子、驚奇、自主漫遊、美感、故事、兔子洞**——一樣是主動投入，只是訴諸情感而非記憶。

## 2. 設計：圖譜為基盤，好奇為層

### 2.1 每個概念的「驚奇迴圈」（單步，無對錯）
```
① 鉤子（curiosity gap）   先拋謎/問句：「AI 為什麼不乾脆全放雲端？」——只引發想知道，不要求作答
② 揭示（aha + 故事 + 生動例 + 雙編碼）  圖上聚焦該節點，用故事與鮮活例子交付一個「喔！」
③ 意外連結（surprise）    「而你絕對想不到——它連到 ___」（萬物相連的愉悅）
④ 邀請漫遊（autonomy/兔子洞） 「想順這條線追下去嗎？」——交還主控權，鼓勵岔路
```

### 2.2 三個模式
- **Wander（漫遊）**：自由探索的喜悅，最大自主；既有圖譜直接支援。
- **Wonders（驚奇之旅）**：策展的「好奇之旅」（原 themes，但定位為*驚奇散步*，不是課）。內容骨架＝邊緣 AI 7 步。
- **Trail（足跡）**：你漫遊過、點亮過的節點在 Deep Field 上發光——**好奇旅程的紀念**（非掌握度），可分享。

### 2.3 進度承載（localStorage）
只記「去過哪些節點」當足跡點亮用。**不記**對錯、不排複習。清快取即失＝對紀念性質可接受。

  "id": "edge-ai", "accent": "TEC",

      "example": { "zh": "自駕車要在毫秒內決定煞不煞…" },  // ② 生動例
      "surprise":{ "zh": "意外的是——它和你大腦的反射弧是同一個道理。" }, // ③ 驚奇連結

- **沒有** check/answer/feedback/card 欄位（評量已移除）。

## 5. 成功標準（情感／好奇導向）

- **主判準**：看完*想不想點下一個*？有沒有至少一個「原來如此！」？*想不想分享*給朋友？
- **次判準**：作品質感拿得出手、呼應 Linku 領域（邊緣/機器人）。
- *不*以記憶留存、答對率、DAU 為準。可找 1~2 個朋友看反應（眼睛有沒有亮）。

## 6. 每個「驚奇之旅」的開工清單

1. 定一句勾人的主題問句（這趟回答什麼好奇）。
2. 選 6~8 概念 + 排序（從直覺起點走到最大「喔！」）。
3. 每概念寫：① 鉤子（誘人問句）② 揭示（故事+至少一個 aha）③ 生動例 ④ 意外連結 ⑤ 漫遊邀請。
   → 這些是新的編輯手藝：**找到那個「喔！」和那條意外連結，是核心功夫**，AI 起草、你定調。
4. outward：寫「被勾住想再往下掉去哪裡」（真實教材/動手，當更深兔子洞）。
5. preview 驗：讀起來順、有驚奇、行動裝置不破版；自己讀完*真的*想點下一個。

## 7. 風險與守則

- **「喔！」與「意外連結」是真功夫**：寫不出驚奇，就只是漂亮的條目。每步至少一個愉悅點，是品質門檻。
- **不可滑回被動閱讀**：主動成分＝好奇鉤子/驚奇/漫遊/美感，必須真的勾人，不是包裝過的維基。
- **範圍守 Gate**：次要資產，每層可停；P1 完成即有可漫遊、有足跡的完整體驗。
- **誠實向外指**：本站點燃好奇、不教技能；想動手的人連到真實資源。
- **半成品不進 master**（自動部署 Cloudflare Pages）：分支完成、E2E 綠燈再合。

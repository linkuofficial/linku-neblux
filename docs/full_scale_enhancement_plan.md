## 優化策略

### 目標
將所有 627 個節點的英文 description 提升到試點節點的專業度水準
- **短期目標**: B 級及以下節點 → 平均 +1.5 分
- **長期目標**: C 級節點 → B 級；B 級節點 → A 級

### 分層方案
1. **P0 (優先級最高)**: C 級節點 (48 個) → 提升至 B 級
   - 耗時: 2-3 週 (batch 化處理)
   - 效益: 品質跨越式提升 + 連鎖改善相關節點

2. **P1 (優先級次高)**: B 級低分節點 (WHAT/SIGNIFICANCE 合計 <8 分): 估計 150-180 個
   - 耗時: 3-4 週 (batch 化處理)
   - 效益: 逐步昇級至 A 級

3. **P2 (優先級低)**: 其餘 B 級節點 (200+ 個)
   - 可視 P1 效果決定是否執行

## 技術規格

### 優化維度 (基於試點成功案例)
1. **WHAT**: 強化開場定義，確保「是什麼」清晰具體
2. **SIGNIFICANCE**: 補充影響力 + 重要性說明
3. **BRIDGE**: 改善跨領域連結表述

### 約束條件
- 語言: 英文 (不涉及翻譯)
- 字數: 保持 [200-400] 字區間 (避免過度擴張)
- 術語: 保留原始關鍵字
- 模型: claude-sonnet-4-6 (已驗證可用)

### 工具鏈
- `scripts/enhance_descriptions.py` (改善 — 已有)
- `scripts/export_enhancement_review.py` (審核表 — 已有)
- `scripts/apply_enhancement_review.py` (套用 — 已有)
- `scripts/quality_check.py` (驗證 — 已有)
- **新增**: `scripts/batch_enhance_full_scale.py` (批量協調)

## 設計（採提案①時）
- **後端 `functions/api/echo.ts`**：
  - `onRequestPost`：body `{tour, step}`；驗 `tour ∈ WONDER_IDS`、`step` 在該 tour 範圍（1-based）；
    同源檢查（`Origin`/`Referer` 比對 host）；D1 `INSERT ... ON CONFLICT(tour,step) DO UPDATE count+1`；
    （岔路 B 若做）KV `ip:tour:step` TTL 3600、>3 忽略（回 200 但不加）。回 `{ok, count}`。
  - `onRequestGet`：`?tour=`：回該 tour 各站計數 `{steps:{[step]:count}}`（或 `{counts:[...]}`）。
  - 零依賴風格（比照 health.ts，不 import workers-types）。CORS 不需要（同源）。
  - `WONDER_IDS` 需後端也有一份——由前端 list 同步一份常數進 function（或 build 時注入；避免漂移）。
- **前端 `wonders-main.js`**：
  - `API_ENABLED` 為真時，載入 tour 後 `GET /api/echo?tour=`（graceful，失敗靜默不顯示 ✨）。
  - surprise beat（`#wp-surprise` 內、`#wp-share` 旁）加安靜 ✨ 鈕＋一行序數小字（三語）。
  - 點 ✨ → `navigator.sendBeacon('/api/echo', {tour,step})` → 樂觀 +1 顯示「你是第 N 位…」。失敗靜默。
  - 鐵律 4：只一行小字、無徽章/點數/彈窗。
- **E2E**：新增守門——`API_ENABLED` 真時 mock `/api/echo` 驗 ✨ 顯示＋點擊 sendBeacon payload；
  **API 掛掉時無 ✨、無 console error**（延用 api-failure 精神）。旗標為 false 時 ✨ 不出現。
